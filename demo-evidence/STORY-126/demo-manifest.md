### AC-001 — SPB Parse: Happy Path (BC-2.01.013 AC-001 / AC-002 / AC-003)

**Behavior demonstrated:** A pcapng containing SHB + IDB(Ethernet) + SPB(14-byte Ethernet frame) is
parsed without panic or error. The SPB packet is attributed to interface 0 (from the preceding IDB).
Timestamp fields are zero (SPB has no per-packet timestamp).

**Fixture:** `spb-happy.pcapng` (80 bytes) — SHB(LE) + IDB(Ethernet, linktype=1) + SPB(14-byte payload)

**CLI command shown:**
```
wirerust summary spb-happy.pcapng --no-color
```

**Expected output:** `WIRERUST TRIAGE REPORT` renders with 0 decoded packets (the Ethernet frame has
no IP layer so packet analysis yields a decode-error skip), no crash, exit code 0.

**Recordings:**

### AC-002 — Block-Skip (F-07): NRB / ISB / SJE Skipped, EPB Parsed (BC-2.01.015)

**Behavior demonstrated:** A pcapng containing SHB + IDB + NRB + ISB + SJE + EPB is read
completely. The three non-packet blocks (NRB, ISB, SJE) are silently skipped (no error,
no logged body bytes per SEC-007). The EPB following them is parsed normally. No crash.

**Fixture:** `spb-skip.pcapng` (140 bytes) — SHB + IDB + NRB(8-byte body) + ISB(empty) + SJE(empty) + EPB(14-byte ARP)

**CLI command shown:**
```
wirerust summary spb-skip.pcapng --no-color
```

**Expected output:** `WIRERUST TRIAGE REPORT` renders, 1 decode-error-skipped packet (ARP frame,
no IP layer), exit code 0. No crash from skip blocks.

**Test evidence for `skipped_blocks` counter (not surfaced in CLI output):**  
`test_BC_2_01_015_dispatch_known_and_skip_unknown` asserts `source.skipped_blocks == 5` (NRB + ISB + SJE + DSB + unknown = 5 skip blocks in that unit test fixture). The CLI recording shows behavioral equivalence (no crash, continued parsing).

**Recordings:**
- `AC-002-block-skip-f07.gif` (158 KB)
- `AC-002-block-skip-f07.webm` (233 KB)
- `AC-002-block-skip-f07.tape` (VHS source)

**ACs traced:**
- BC-2.01.015 AC-006: NRB / ISB / SJE / DSB have explicit named skip arms (F-07 MANDATORY)
- BC-2.01.015 AC-008: no block body bytes logged; DSB body not emitted (SEC-007)
- BC-2.01.015 AC-010: skipped_blocks counter incremented (verified in unit test)

### AC-003 — Error Surface: SPB Before Any IDB → E-INP-009 (BC-2.01.017 PC1)

**Behavior demonstrated:** A pcapng containing SHB + SPB (no IDB) triggers a graceful error.
The full anyhow error chain is displayed on stderr, containing the exact BC-2.01.017 PC1
context string. No panic; no silent failure.

**Fixture:** `spb-error.pcapng` (48 bytes) — SHB(LE) + SPB(4-byte payload, no IDB before it)

**CLI command shown:**
```
wirerust summary spb-error.pcapng --no-color
```

**Expected output (exact):**
```
Error: Failed to read spb-error.pcapng

Caused by:
    pcapng Simple Packet Block encountered before any Interface Description Block: SPB encountered but interface table is empty — no IDB has been parsed (E-INP-009)
```

The recording shows all three required substrings:
1. `"pcapng Simple Packet Block encountered before any Interface Description Block"` (BC-2.01.017 PC1 context)
2. `"SPB encountered but interface table is empty — no IDB has been parsed"` (BC-2.01.013 PC5 exact message)
3. `"E-INP-009"` (error taxonomy code)

**Recordings:**
- `AC-003-spb-before-idb-e-inp-009.gif` (169 KB)
- `AC-003-spb-before-idb-e-inp-009.webm` (243 KB)
- `AC-003-spb-before-idb-e-inp-009.tape` (VHS source)

**ACs traced:**
- BC-2.01.017 PC1: error has context string "encountered before any Interface Description Block"
- BC-2.01.013 AC-001 (error path): SPB before IDB → Err(E-INP-009), not Ok
- BC-2.01.017 AC-012: no panic on malformed/mis-ordered pcapng
